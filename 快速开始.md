# 🚀 快速开始 - 一键打包指南

## 📦 已生成的文件

| 文件名 | 平台 | 说明 |
|--------|------|------|
| `build-win.bat` | Windows | 完整打包脚本（生成安装包） |
| `build-mac.sh` | macOS/Linux | 完整打包脚本（生成安装包） |
| `build-portable-win.bat` | Windows | 生成单文件可执行程序 |
| `build-portable-mac.sh` | macOS/Linux | 生成单文件可执行程序 |

## ⚡ Windows 快速开始

### 方法 1: 生成安装包（推荐）
直接双击运行 `build-win.bat`

### 方法 2: 生成单文件可执行程序
直接双击运行 `build-portable-win.bat`

输出位置：
- 安装包: `src-tauri/target/release/bundle/msi/*.msi`
- 单文件: `release-portable/mmy-tasktime.exe`

## 🍎 macOS/Linux 快速开始

### 首次使用（只需执行一次）
```bash
chmod +x build-mac.sh
chmod +x build-portable-mac.sh
```

### 生成安装包
```bash
./build-mac.sh
```

### 生成单文件可执行程序
```bash
./build-portable-mac.sh
```

输出位置：
- macOS: `src-tauri/target/release/bundle/dmg/*.dmg`
- Linux: `src-tauri/target/release/bundle/appimage/*.AppImage`
- 单文件: `release-portable/`

## 📋 前置要求

### Windows
- ✅ Node.js (v18+)
- ✅ Rust + Cargo
- ✅ WebView2 Runtime (Win11 已内置)

### macOS
- ✅ Node.js (v18+)
- ✅ Rust + Cargo
- ✅ Xcode Command Line Tools

### Linux
- ✅ Node.js (v18+)
- ✅ Rust + Cargo
- ✅ WebKitGTK 等系统依赖

## ❓ 常见问题

**Q: 提示 "node" 不是内部命令？**
→ 安装 Node.js: https://nodejs.org/

**Q: 提示 Rust/Cargo 未找到？**
→ 安装 Rust: https://rustup.rs/

**Q: 打包很慢？**
→ 首次打包需要下载依赖，耐心等待 5-10 分钟

**Q: 如何生成真正的单文件（无需安装）？**
→ 使用 `build-portable-*.bat/sh` 脚本

## 🎯 推荐使用流程

1. **开发测试**: 使用 `npm run dev` 启动开发模式
2. **本地打包**: 运行对应平台的打包脚本
3. **测试安装包**: 在虚拟机或另一台电脑上测试
4. **分发**: 将 `bundle/` 目录下的安装包分发给用户

---

**提示**: 如果是首次打包，建议先运行完整打包脚本测试流程，再使用便携版脚本生成单文件。
