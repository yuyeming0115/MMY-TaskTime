# 桌面端问题复盘

记录 2026-06-29 这轮桌面端测试中暴露的问题、根因和后续约定，用于避免同类问题反复出现。

## 1. 透明边框和窗口尺寸

现象：
- 完整模式或精简模式边缘出现透明区域。
- 从精简模式恢复完整模式后，UI 偶尔变窄。
- 完整模式手动调整窗口后，透明区域再次出现。

根因：
- Tauri 窗口使用 `transparent: true` 和 `decorations: false`，窗口比内容大时会直接露出透明 WebView。
- 之前在 `renderAll()` 中按内容尺寸反复调用 `set_window_size`，和用户手动拖拽缩放互相抢控制权。
- 旧版用 `outer-inner` 差值补偿窗口尺寸，在无边框透明窗口中反而会把窗口撑大。

约定：
- 完整模式容器必须填满 Tauri 窗口，不能用固定内容宽高去适配外层窗口。
- 完整模式允许用户缩放，JS 不在常规渲染中自动改完整模式窗口尺寸。
- 精简模式才使用固定尺寸，并在切换时主动设置窗口大小。
- 默认完整模式尺寸为 `350x600`，旧默认尺寸缓存需要迁移。

## 2. 模式切换导致任务终止

现象：
- 展开和精简切换后，常驻任务或自定义任务全部停止。

根因：
- 新增自定义任务或切换相关流程曾调用 `initTasks()` 重建 `tasks`，导致运行态丢失。
- Tauri 窗口变化可能触发 WebView 状态波动，运行态只存在内存里不够稳。

约定：
- 新增自定义任务只增量添加任务状态，不重建全部任务。
- 需要重建任务列表时，先保存同 ID 任务的 `running/remaining/isPrimary`，重建后再迁移回来。
- 运行态使用单独的 `mmy_tasktime_runtime` 保存，包含当前模式、完整模式尺寸、运行任务和剩余时间。

## 3. 前端资产目录

现象：
- `npm run build` 或 `cargo check` 报错，提示 `frontendDist` 包含 `src-tauri/target`、`node_modules` 或锁文件。

根因：
- `frontendDist` 曾配置为 `../`，Tauri 会把项目根目录整体当成前端资产扫描。

约定：
- 使用 `dist/` 作为唯一前端产物目录。
- `npm run prepare:dist` 只复制 `index.html`、`style.css`、`app.js`。
- `tauri.conf.json > build.frontendDist` 固定为 `../dist`。

## 4. UI 设计经验

完整模式：
- 顶部使用左右分栏专注面板：左侧真实模拟时钟，右侧主任务倒计时。
- 主任务进度使用整块透明填充，不使用过细进度条。
- 常驻任务按钮直接显示运行中剩余时间，避免顶部运行条重复展示常驻任务。
- 常驻任务按钮文字必须居中，并和区块标题保持足够距离。

精简模式：
- 默认跟随浅色主题，避免在浅色环境里显得像反色。
- 黑白模式通过底部 `◐/◑` 按钮切换，完整模式和精简模式共用同一个主题设置。
- 精简模式显示“下个到点”信息，避免底部空白。

设置面板：
- 常驻任务开放 4 个固定槽位编辑：名称、分钟数、是否循环。
- 保留固定 ID（`q5/q10/q30/q40`），避免破坏托盘快捷入口和历史运行逻辑。
- 保存常驻任务设置后立即生效，并尽量保留同 ID 的运行状态。

## 5. 发布检查

发布前至少执行：

```bash
node --check app.js
npm run build
```

单文件可执行程序来自：

```text
src-tauri/target/release/mmy-tasktime.exe
```

对外交付时复制到：

```text
releases/MMY-TaskTime-v版本号.exe
```
